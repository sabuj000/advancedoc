# User Documentation Style (Microsoft Style–aligned)

**Goal:** Clear, inclusive guidance for people completing tasks in the product.

## Voice and tone

- Use **second person** (“you”).
- Use **imperative** verbs for steps (“Select…”, “Enter…”, “Open…”).
- Prefer **short sentences** and **simple words**; avoid jargon unless the UI uses it.

## Structure

- Start with **what the reader will achieve** (overview).
- List **prerequisites** before steps.
- Use **numbered steps**; one primary action per step.
- End with **result** and optional **additional information**.

## Inclusive and precise language

- Name UI elements **exactly** as labeled (casing, punctuation).
- Avoid blame (“obviously”, “just”); state facts.
- When multiple roles exist, say **who** the guide is for in the overview.

## Prohibited

- Passive voice where it hides the actor (“it is recommended” → “We recommend” or imperative).
- Placeholder steps without product-specific detail.
