# KB Article — Checklist

- [ ] **Title** is specific and searchable (includes error or symptom keywords where helpful).
- [ ] **Scope** states what is and is not covered.
- [ ] **Prerequisites** list tools, roles, or versions needed.
- [ ] **Problem Statement** uses observable facts (messages, HTTP codes, UI behavior).
- [ ] **Solution** summarizes the fix before steps.
- [ ] **Steps to Follow** are numbered, imperative, and ordered.
- [ ] **Expected Outcome** describes how you confirm success.
- [ ] **Additional Resources** link to related guides or API docs when relevant.
- [ ] Second person and imperative voice throughout (`style/user-style.md`).
- [ ] No vague steps (“fix the config” without naming the setting).
