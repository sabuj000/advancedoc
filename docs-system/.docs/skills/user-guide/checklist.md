# User Guide — Checklist

- [ ] **Title** matches the user task (“How to …”).
- [ ] **Overview** explains outcome and audience in plain language.
- [ ] **Prerequisites** are complete (roles, data, prior steps).
- [ ] **Steps** are numbered; each step is one main action.
- [ ] UI labels match the product; API field names match public docs or OpenAPI.
- [ ] **Result** states observable success criteria.
- [ ] **Additional Information** covers limits, tips, or links without duplicating the steps.
- [ ] Tone follows `style/user-style.md` (second person, clear, concise).
