# Skill: User Guide

## Purpose

Create user-facing procedural documentation: clear overview, prerequisites, numbered steps, and explicit success criteria.

## When to Use

- Teaching a feature end-to-end for customers or internal users.
- Onboarding flows that mirror the product UI or public API from a user perspective.

## Steps

1. **Clarify the feature** — One-sentence goal and who it is for.
2. **List prerequisites** — Accounts, permissions, data, or prior tasks.
3. **Write steps** — One action per step; name UI controls and fields as they appear.
4. **Describe result** — What “done” looks like.
5. **Apply template** — Use `templates/user-guide.md`; validate with `checklist.md`.

## Outputs

- Markdown file under `user-docs/guides/` with kebab-case filename.
